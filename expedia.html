This is a usability report conducted on travel website Exepdia. We conducted a competitive analysis of the existing products features in different platforms.




Stage 1
Identified basic tasks a user would perform on Expedia (mobile, website).

Stage 2
Defined scenarios and tasks based on the findings.

Stage 3
Created Personas.

Stage 3
Conducted Cognitive Walkthrough & Heuristics on both platforms.

Stage 4
Drafted a test script that encompasses every scenario and task.

Stage 5
Conducted a pilot run.Stage 6: Recruit participants to participate in user evaluations.

Stage 6
Recruited participants to participate in user evaluations.

Stage 7
Conducted user evaluations.

Stage 8
Analyzed video recordings and transcribe user quotes for qualitative data.

Stage 9
Analyzed video recordings to gather quantitative data such as number of clicks, time on task etc.

Stage 10
Created an Expert Usability Evaluation Report with detailed explanation of the issues encountered and recommendations.

User Demographics and Tasks

A total of 8 users were requested to evaluate the application based on 9 fundamental tasks identified previously from the expert review.

Quantitative Measures

A range of qualitative measures were considered such as

Θ Time on task
Θ Task Completion
Θ Pre Test Questionnaire
Θ Post Test Questionnaire
Θ System Usability Scale

It was observed that the Time on Task the mobile was greater than the website, even though users liked the website better. We used the SUS scale to quantify user opinions.

Overview of Issues and Recommendations

I identified issues that plagued both the platforms. User evaluation revealed, majority of users had the same pain points as the others. This consistent trend helped me rate the priority of these global issues as critical.

Issue: Excessive Loading Time

 

Issue: Non Responsive
